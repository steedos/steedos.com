这是一个用于显示、新建或编辑对象记录的表单组件。

## 基本用法

最基本的用法是配置组件的“对象”及“记录ID”属性，就可以显示出某条对象记录的只读表单页面。

如下所示我们配置对象表单的“对象”属性为当前对象，“记录ID”属性为当前记录：

```json
{
    "type": "page",
    "title": "Welcome to Steedos",
    "body": [
        {
            "type": "steedos-object-form",
            "label": "对象表单",
            "objectApiName": "${objectName}",
            "recordId": "${recordId}",
            "className": "sm:border sm:shadow sm:rounded sm:border-gray-300 bg-white p-4"
        }
    ]
}
```
如果我们按照教程 [Amis微页面 - 记录页面](/docs/amis/page-type#%E8%AE%B0%E5%BD%95%E9%A1%B5%E9%9D%A2) 新建一个指向人员对象的微页面，并拖动一个对象表单组件到画布，可以看到该组件生成了类似上面的amis Schema配置。

发布后，我们就可以在人员对象的记录详细页面看到只读的人员记录表单效果。

objectApiName属性可以被配置为像`space_users`或`accounts`这种固定值，也可以配置为变量，而变量`objectName`在“记录页面”、“列表页面”以及“表单”类型的微页面中存储了当前对象Api名称。

recordId属性类似的可以配置为某个具体的记录id值，但是一般我们会配置为变量，比如在“记录页面”或“表单”类型的微页面中配置为当前记录Id值。

## 只读/可编辑

对象表单 ObjectForm 组件默认是只读的，一般用于在“记录页面”的微页面中配置一个只读的对象表单。

可以通过修改其mode属性为“eidt”来把对象表单组件显示为可编辑状态。

## 字段

在设计球中选中对象表单后，右侧属性选项卡中可以使用[显示的字段](#显示的字段)，[排除的字读](#排除的字读)和[重写字段配置](#重写字段配置)功能来实现对象表单字段的自定义。

### 显示的字段

可以选择想要显示的字段，被选中的字段会在表单中显示。

点击**显示的字段**输入框，在弹出框中选中想要显示的字段即可。

当然，也可以在代码中直接配置`fields`实现字段的显示，代码示例如下：
```json
{
  "type": "steedos-object-form",
  "label": "对象表单",
  "objectApiName": "organizations",
  "recordId": "6448fd24ad9a40d1589b7058",
  "fields": [
    "name",
    "parent",
    "modified_by"
  ]
}
```
上述示例演示了显示`name`、`parent`和`modified_by`字段。

### 排除的字段

可以排除不想要的字段，被选中的字段不会在表单中显示。

点击**排除的字段**输入框，在弹出框中选中想要排除的字段即可。

当然，也可以在代码中直接配置`excludedFields`实现字段的排除，代码示例如下：
```json
{
  "type": "steedos-object-form",
  "label": "对象表单",
  "objectApiName": "organizations",
  "recordId": "6448fd24ad9a40d1589b7058",
  "excludedFields": [
    "parent"
  ]
}
```
上述示例演示了排除`parent`字段。

<alert type="info">
    如果显示的字段和排除的字段同时选中同一个字段，则该字段不会显示，排除的字段优先级高于显示的字段。
</alert>

### 重写字段配置

可以重写字段的配置，**重写字段配置**代码框中格式为`json`，代码示例如下：
```json
{
    "name": {               // 配置需要重写字段的API名称
        "is_wide": true,    // 设置为宽字段
        "required": true,   // 设置为必填
        "amis": {           // 设置渲染属性
            "type": "input-color" // 将字段重写为 颜色选择器
        } 
    }, 
    "title": {          // 配置需要重写字段的API名称
        "group": "分组1" // 设置分组
    }
}
```

## 接口

编辑状态下的对象表单，初始化或保存数据时，发送或接收的数据不符合接口要求，可以使用初始化数据接口或保存数据接口来实现。设计器选中表单后，右侧**接口**选项卡可以使用该功能。

请求接口前会执行发送适配器，接收到传回的数据会执行发送适配器。

[了解更多关于发送适配器](https://aisuda.bce.baidu.com/amis/zh-CN/docs/types/api?page=1#配置请求适配器)。  
[了解更多关于接收适配器](https://aisuda.bce.baidu.com/amis/zh-CN/docs/types/api?page=1#配置接收适配器)。

<alert type="info">
    只读状态 的对象表单无法使用初始化数据接口和保存数据接口功能。
</alert>

### 初始化数据接口

进入表单后获取表单的初始数据。

在初始化数据时，若无法配置出想要的请求结构，请在初始化接口的发送适配器进行配置，配置后代码中会有`initApiRequestAdaptor`属性，
[查看initApiRequestAdaptor源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/lib/converter/amis/api.js#L227)。

在`api.data`中添加一个`test`属性，示例如下：
```json
{
  "type": "steedos-object-form",
  "label": "对象表单",
  "objectApiName": "project",
  "recordId": "630486ce9cc40f03d6e7cb60",
  "className": "sm:border sm:shadow sm:rounded sm:border-gray-300 bg-white p-4",
  "mode": "edit",
  "initApiRequestAdaptor": "api.data['test'] = 'initApiRequestAdaptor的测试';\nconsole.log('api.data ==> ', api.data);\nreturn api;" // 初始化接口的请求适配器
}

/*
    api.data ==> {
        test: "initApiRequestAdaptor的测试"
        query: "{data:project(filters:[\"_id\", \"=\", \"630486ce9cc40f03d6e7cb60\"]){_id,account,closed,contracts_all_income,contracts_all_pay,contracts_made_invoice,contracts_paid_pay,contracts_received_income,contracts_received_invoice,contracts_waiting_income,contracts_waiting_pay,deadline,description,internal,kickoff,name,open_issues,open_tasks,others,project_code,project_cost_budget,project_labor,project_labor_balance,project_labor_budget,project_manager,project_program,project_type,status,business_cost_budget,business_pay,daily_cost_budget,daily_pay,_display:_ui{account,closed,contracts_all_income,contracts_all_pay,contracts_made_invoice,contracts_paid_pay,contracts_received_income,contracts_received_invoice,contracts_waiting_income,contracts_waiting_pay,deadline,internal,kickoff,open_issues,open_tasks,others,project_cost_budget,project_labor,project_labor_balance,project_labor_budget,project_manager,project_program,project_type,status,business_cost_budget,business_pay,daily_cost_budget,daily_pay}, \n    recordPermissions: _permissions{\n        allowCreate,\n        allowCreateFiles,\n        allowDelete,\n        allowDeleteFiles,\n        allowEdit,\n        allowEditFiles,\n        allowRead,\n        allowReadFiles,\n        disabled_actions,\n        disabled_list_views,\n        field_permissions,\n        modifyAllFiles,\n        modifyAllRecords,\n        modifyAssignCompanysRecords,\n        modifyCompanyRecords,\n        uneditable_fields,\n        unreadable_fields,\n        unrelated_objects,\n        viewAllFiles,\n        viewAllRecords,\n        viewAssignCompanysRecords,\n        viewCompanyRecords,\n      }\n    }}"
    }
*/
```

`initApiRequestAdaptor`中的代码：
```js
api.data['test'] = 'initApiRequestAdaptor的测试';
console.log('api.data ==> ', api.data);
return api;
```

同样的，后端返回的响应结构无法满足接口格式要求，请在初始化接口的接收适配器进行配置，配置后代码中会有`initApiAdaptor`属性，
[查看initApiAdaptor源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/lib/converter/amis/api.js#L298)。

在 `payload` 中添加一个 `test` 属性，示例如下：
```json
{
  "type": "steedos-object-form",
  "label": "对象表单",
  "objectApiName": "project",
  "recordId": "630486ce9cc40f03d6e7cb60",
  "className": "sm:border sm:shadow sm:rounded sm:border-gray-300 bg-white p-4",
  "id": "u:7fd7df80dcbc",
  "mode": "edit",
  "initApiAdaptor": "payload[\"test\"] = 'initApiAdaptor的测试';\nconsole.log('payload ==> ', payload);"  // 初始化接口的接收适配器
}

/*
    payload ==> {
        test: "initApiAdaptor的测试"
        data: {account: 'ua7esWTyhgNaspC94', closed: false, contracts_all_income: 6000000, contracts_all_pay: 7300000, contracts_made_invoice: 0, …}
    }
*/
```

`initApiAdaptor` 中的代码：
```js
payload["test"] = 'initApiAdaptor的测试';
console.log('payload ==> ', payload);
```


### 保存数据接口

点击保存按钮后会保存表单中的数据。

在初始化数据时，若无法配置出想要的请求结构，请在保存数据接口的发送适配器进行配置，配置后代码中会有`apiRequestAdaptor`属性。
[查看apiRequestAdaptor源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/lib/converter/amis/graphql.js#L235)。

同样的，后端返回的响应结构无法满足接口格式要求，请在保存数据接口的接收适配器进行配置，配置后代码中会有`apiAdaptor`属性。 
[查看apiAdaptor源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/lib/converter/amis/api.js#L327)。


## 底部按钮栏

当对象表单 ObjectForm 组件被配置为可编辑时，默认是不带提交按钮的。

如果需要在表单底部显示提交按钮，只要配置 `actions` 属性即可，这是amis原生能力，具体请参阅文档 [底部按钮栏](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index#%E5%BA%95%E9%83%A8%E6%8C%89%E9%92%AE%E6%A0%8F)。

```json
{
    "type": "page",
    "title": "Welcome to Steedos",
    "body": [
        {
            "id": "test",
            "type": "steedos-object-form",
            "objectApiName": "organizations",
            "recordId": "gKfnkfbLWdqCxo8dg",
            "mode": "edit",
            "actions": [
                {
                    "type": "button",
                    "label": "保存",
                    "actionType": "submit",
                    "level": "info"
                }
            ]
        }
    ]
}
```

要提交表单，只要像上面示例一样，在 `actions` 节点配置一个 `submit` 按钮即可，而不用费力配置 api 请求接口。

除了可以轻松实现提交表单外，还可以定制表单提交后行为，这些功能与amis原生表单组件没有区别，请参阅 [amis Form 表单提交后行为](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index#%E6%8F%90%E4%BA%A4%E5%90%8E%E8%A1%8C%E4%B8%BA)。

## 触发表单事件

对象表单 ObjectForm 组件最终输出的是一个 [amis Form 表单](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index) 组件，我们可以像使用 amis 原生表单组件 一样实现跨组件触发表单事件。

以下示例演示了跨组件提交表单，只要正确的配置组件的`id`属性以及事件的`componentId`属性即可。

```json
{
    "type": "page",
    "title": "Welcome to Steedos",
    "body": [
        {
            "id": "test",
            "type": "steedos-object-form",
            "objectApiName": "organizations",
            "recordId": "gKfnkfbLWdqCxo8dg",
            "mode": "edit"
        },{
            "type": "button",
            "label": "提交上面的表单",
            "onEvent": {
            "click": {
                    "actions": [
                        {
                            "componentId": "test",
                            "actionType": "submit"
                        }
                    ]
                }
            }
        }
    ]
}
```

## 弹出表单

在微页面设计器中拖动一个普通的“按钮”组件到画布，然后在设计器中为其添加一个“打开弹窗”的动作，在“弹框内容”画布中拖动一个华炎魔方“对象表单”，配置好相关属性后就可以轻松实现弹出对象表单功能。

最后设计器会生成类似如下内容的 amis Schema 配置。

```json
{
    "type": "page",
    "title": "Welcome to Steedos",
    "body": [{
        "type": "button",
        "label": "按钮",
        "onEvent": {
        "click": {
            "actions": [{
                "actionType": "dialog",
                "dialog": {
                "type": "dialog",
                "title": "弹框标题",
                "body": [
                    {
                        "type": "steedos-object-form",
                        "label": "对象表单",
                        "objectApiName": "organizations",
                        "recordId": "623NR6NCZJP8irB4y",
                        "mode": "edit"
                    }
                ],
                "size": "xl"
            }]
        }
    }]
}
```

## 属性表

除了支持 [amis Form 表单属性表](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index#%E5%B1%9E%E6%80%A7%E8%A1%A8) 中的配置以外，还支持下面的配置。

| 属性名 | 类型 | 默认值 | 说明 |
|----|----|----|----|
| objectApiName | string | space_users | 关联对象api名称 |
| recordId | string | 无 | 关联记录ID值 |
| mode | string | read | 只读read/编辑edit |
| layout | string | normal | 表单项布局，同amis form的mode属性 |
| fields | array | 无 | 显示的字段 |
| excludedFields | array | 无 | 排除的字段 |
| fieldsExtend | string | 无 | 重写字段配置 |

## 事件表

支持 [amis Form 表单事件表](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index#%E4%BA%8B%E4%BB%B6%E8%A1%A8) 中的所有配置。

## 动作表

支持 [amis Form 表单动作表](https://aisuda.bce.baidu.com/amis/zh-CN/components/form/index#%E5%8A%A8%E4%BD%9C%E8%A1%A8) 中的所有动作。

另外所有的对象表单组件只读模式都监听了对象的[`@data.change`事件](/docs/amis/component#@data.change事件)，只要触发对象的`@data.change`事件，就能间接调用该组件的以下动作： 
- [刷新](#刷新)
- [跳转到列表页面](#跳转到列表页面)

详情请参考 [对象表单组件只读模式源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/@steedos-widgets/amis-lib/src/lib/converter/amis/index.js#L426)。  

### 刷新

以下示例演示点击按钮后，刷新名为`project`的对象表单组件。

```json
{
  "type": "button",
  "label": "刷新对象表单组件",
  "onEvent": {
    "click": {
      "actions": [
        {
          "actionType": "broadcast",
          "args": {
            "eventName": "@data.changed.project"
          },
          "data": {
            "objectName": "project"
          }
        }
      ]
    }
  }
}
```

在只读模式下，点击上述示例中的按钮后会触发绑定`project`对象的对象表单组件的`@data.changed`事件，在该事件中，会根据传入的参数判断是否刷新对象表单组件。

要刷新对象表单组件，需在`data`中配置以下参数：
- objectName: `String`。要触发广播事件对象的对象Api名称，即列表视图绑定的对象api名称。

在华炎魔方记录详情页中编辑记录后，会刷新只读页面中的对象表单组件，就是因为在弹出的编辑表单组件中监听了提交成功事件并触发了页面中只读的对象表单组件的`@data.change`事件，在`@data.change`事件中会执行刷新动作。
[对象表单组件源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/lib/converter/amis/index.js#L350)。

### 跳转到列表页面

在华炎魔方记录详情页中删除记录后，会跳转列表页面页面，就是因为触发了点击事件并触发了页面中只读的对象表单组件的`@data.change`事件，在`@data.change`事件中会执行跳转动作。

详情请参考 [删除按钮源码](https://github.com/steedos/steedos-widgets/blob/1.2/packages/%40steedos-widgets/amis-lib/src/schema/standard_delete.amis.js#L44)。
